WSL2 — Windows Subsystem for Linux

Запускайте настоящий Linux внутри Windows

ДО WSL2 (проблемы)

  • Cygwin/MinGW — ломанный Linux
  • Git Bash — урезанный shell
  • Docker Toolbox — медленный
  • Проблемы с CRLF/LF

С WSL2 (решения)

  • Настоящий Ubuntu/Debian/Fedora
  • Полный bash/zsh/fish
  • Docker Desktop с WSL2 backend — быстро!
  • Нативная работа с LF
  • Symlink'и работают!

⚡ Быстрая установка WSL2:

# Одна команда:
wsl --install
# Выбор дистрибутива:
wsl --install -d Ubuntu-24.04

Shell: Bash vs Zsh vs Fish

Bash

СТАНДАРТ
Совместимость: работает ВЕЗДЕ и ВЕГДЕ
Скрипты работают везде
Простота — легко понять основы

По умолчанию: Ubuntu/Debian • WSL • macOS (до Catalina)

Zsh

РЕКОМЕНДУЕТСЯ!
Расширенное globbing (**/*.py рекурсивно)
Smart tab completion (меню, fuzzy поиск)

Fish

FRIENDLY!
Autosuggestions — подсказки как в IDE

Web-based config (fish_config)
Tab completion из примеров

Идеально для интерактивного daily use

PowerShell 7 — Cross-Platform

Object-Oriented Pipeline: вместо текста — объекты .NET передаются между командами. Работает на macOS и Linux! Отлично для Azure/OFFICE автоматизации.

Get-Process | Sort-Object CPU -Descending | Select-Object -First 10

ПРАКТИЧЕСКОЕ ЗАДАНИЕ

Учебная практика: Аналитический отчёт

Индивидуальное исследование и обоснованный выбор ПО разработчика

Цель задания

Провести сравнительный анализ популярных инструментов разработки в каждой категории, выбрать оптимальный набор ПО для своей рабочей станции и обосновать выбор инженерными аргументами.

НЕ «мне нравится»
А «выбираю X, потому что для моих задач Y важно, а Z не подходит из-за...»
НЕ копировать
У каждого студента должен быть СВОЙ уникальный стек
Сравнивать
Минимум 3 варианта в каждой категории с таблицами

1. Исследовать

Изучить топ-ПО в каждой категории: характеристики, преимущества, недостатки, системные требования, лицензирование. Использовать данный гайд как источник.

2. Сравнить

Построить таблицы сравнения по критериям: производительность, удобство, экосистема, цена, сообщество, совместимость с вашей ОС.

3. Обосновать

Сделать выбор и аргументировать: почему этот инструмент лучше подходит именно для ваших задач, бюджета, оборудования и рабочего процесса.

📋 6 категорий для анализа (обязательно все):

1. Операционная система

Выбрать ОС для разработки

Варианты: Windows 11 (+WSL2), macOS (Intel/Silicon), Linux (Ubuntu/Fedora/Arch/Pop!_OS)

2. IDE / Редактор кода

Основной инструмент написания кода

Варианты: VS Code, JetBrains (WebStorm/PyCharm/etc.), Vim/Neovim, Sublime Text

3. Терминал + Shell

Командная строка

Терминал: WT/iTerm2/Kitty • Shell: Bash/Zsh/Fish/PowerShell

4. Git-клиент

Работа с системой контроля версий

Варианты: Git CLI, GitHub Desktop, GitKraken, SourceTree, LazyGit

5. Контейнеры (Docker)

Изоляция окружений

Docker Desktop (Mac/Win) или Docker Engine (Linux native)

6. AI-ассистент (опционально)

Ускорение разработки

Варианты: GitHub Copilot, Cursor IDE, ChatGPT/Claude

Что должно получиться?

📄 Документ (отчёт):

  • 15-25 страниц структурированного анализа
  • 6+ сравнительных таблиц по критериям
  • Скриншоты установки и настройки выбранного ПО
  • Конфигурационные файлы (settings.json, .zshrc, etc.)
  • Обоснование выбора в каждой категории

💻 Практический результат:

  • Настроенная рабочая станция с выбранным стеком
  • Понимание ПОЧЕМУ выбран именно этот инструмент
  • Навык сравнительного анализа ПО
  • Умение аргументировать технические решения
  • Портфолио для демонстрации работодателю

Критерии оценки

«Отлично» (90-100) 5

Глубокий анализ всех 6 категорий, детальные таблицы (5+ критериев), уникальное обоснование с привязкой к личным задачам, скриншоты настройки, конфиги в приложениях, рефлексия о процессе выбора.

«Хорошо» (75-89) 4

Все категории покрыты, корректные таблицы сравнения, обоснование присутствует но может быть общим («хороший инструмент»), скриншоты есть.

«Удовлетворительно» (60-74) 3

Все категории рассмотрены, но сравнение поверхностное (2-3 критерия), обоснование слабое («нравится», «популярный»), мало скриншотов.

«Неудовлетворительно» (<60) 2

Не все категории покрыты, нет сравнительных таблиц, выбор не обоснован, работа неполная или скопирована у одногруппника.
